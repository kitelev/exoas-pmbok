# R11 — Supply chain compromise (nanotar npm)

Если nanotar npm package compromised (account takeover, typosquatting, dep injection), malicious version may install with `^0.3.0` semver. Plugin then executes arbitrary code on user devices via tarball extraction path.

Mitigation: exact pin (no semver range) prevents auto-upgrade. CI audit detects known CVEs. Vendoring option keeps full control if supply chain confidence low.
