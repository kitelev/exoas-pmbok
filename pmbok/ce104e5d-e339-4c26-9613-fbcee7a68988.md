## 2. Business Case

Parent RFC `b6ba5595` v2 shipped declarative TBox `exo__FocusProfile` + 4 profile-* instances (Phase 1+2+3+6 closed AllAccepted 2026-05-31). Однако **Phase 4 plugin runtime explicitly OUT OF SCOPE** того RFC — declarative artifacts inert. SPARQL queries return all assets regardless of active profile. No command palette UX для switching. No settings UI для PAT, profile selection, cache management.

Phase 4 v3 ships plugin runtime в **backward-compat mode** (Vision Lock #18 scope split): 1 new TBox property + plugin TypeScript code (~1500-2000 LOC) делающий FocusProfile artifacts operational. Phase C+D (full submodule abandonment + iPhone bootstrap) deferred к future RFC pending disk cleanup + plugin maturity.

**Why now:** 3-agent review (Architect / Security / Migration) identified 17 risks с full mitigation strategy. Vision Lock #16 (cross-vault TBox overlay) prevents iPhone broken-class-refs in future Phase C+D. Vision Lock #17 (TS-floor) prevents plugin self-brick. Scope split removes Migration BLOCKERs (M1-M4) from immediate critical path while still shipping incremental value (RDF-level filter + PAT secure handling + command palette UX).
