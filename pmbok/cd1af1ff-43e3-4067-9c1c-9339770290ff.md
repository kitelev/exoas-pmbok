# R17 — TarExtractor memory pressure on large AS

Naive implementation buffers entire tarball ArrayBuffer + all extracted file contents в RAM. iOS plugin sandbox has tight mem limits (~100-200MB working set). Large AS (50MB+ tarball, hundreds of files) → OOM kill + plugin crash.

Mitigation: B.2 streaming AsyncIterable yields entries one at a time. Caller pipes к disk immediately. Peak memory bounded by largest single file, not total tarball size.
