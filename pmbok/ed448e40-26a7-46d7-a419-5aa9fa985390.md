## 2. Business Case

`apply`/`GroundingExecutor` уже покрыты TS-тестами, но эти тесты проверяют код движка, не **определения-команд-как-данных** (51 `exocmd__Command` инстансов в exocmd assetspace). Регрессии в command-definitions (grounding spec, precondition, ассет-шаблон) не ловятся существующими TS-тестами. Нет механизма для CQ «какие команды без тестов». Этот проект внедряет homoiconic ABox-driven testing + coverage queries.

## 3. Цели / MVG (SMART)

- **Specific:** byte-for-byte ABox-driven testing pilot для `bb00efed` (Create Task Instance) + grounding `a6ef8fda`
- **Measurable:** CI зелёный на pilot test-case + красный на intentional corruption + CQ5a/CQ5b возвращают coverage gap для 51 команды
- **Achievable:** 35-70h net (с prior-art reuse из `command-pilot.integration.test.ts`)
- **Relevant:** прямо решает homoiconicity invariant gap (Q1=Да для тест-кейс-конфигурации)
- **Time-bound:** delivery 2026-06-27 (M5)

## 4. Scope (см. WBS Project body)

**IN:**
- 6 phases: CLI determinism / Assetspace setup / Ontology / Repo split / CI workflow / Hooks patch
- Pilot scope: 5 классов + 9 properties + 3 status named-individuals + 1 SHACL shape + 2 new repos
- Pilot command: bb00efed + grounding a6ef8fda (TaskPrototype + 3 InheritanceRule)

**OUT (FUTURE / Phase 5):**
