# R2 — Wikilink re-quoting при копировании

YAML parse теряет outer quotes у namespace-prefixed wikilinks (`"[[32d2374c-1bef-4b64-ad23-e26b53b52df8]]"` → `[[32d2374c-1bef-4b64-ad23-e26b53b52df8]]` после `metadataCache` parse). Если copy-loop пишет value 1:1 — YAML emit может интерпретировать `!` как tag directive, генерируя broken frontmatter.

**Mitigation precedent:** existing `inheritParentContext` использует `formatWikilink` именно для этой проблемы (issue #2850). Re-use того же helper.
