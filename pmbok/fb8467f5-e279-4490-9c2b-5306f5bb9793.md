## 2. Business Case

Phase 4 shipped soft switch only (RDF graph filter via `NoteToRDFConverter.convertVault(effectiveOntologies)`). This delivers cognitive load reduction + privacy boundary, **но НЕ iPhone reindex perf gain** (10min reindex pain persists). Phase 5 hard switch = real prize delivery: filesystem mutation → 10min iPhone reindex → ≤2min.

Beyond perf: hard switch enables real disk footprint reduction (submodule .git directories no longer occupy MB-GB for inactive profiles) + correct cross-device profile sync model (single-active per device honored via auto-reconcile).

## 3. Цели / SMART

- Hard switch functional: vault filesystem reflects effective set после switch (manual `ls assetspaces/` matches effective ontology UIDs)
- iPhone reindex ≤ 2 min для profile-mobile (5-AS effective set) vs current 10 min — **THE KEY METRIC**
- Atomicity: induce mid-Phase 2 failure → vault state either fully old-profile OR fully new-profile, never partial
- Cache restoration: hard switch к recently-switched-from profile (cache hit) completes ≤ 5 sec (no API call)
- Cross-device sync: Mac switch commits vault → iPhone pulls → iPhone auto-reconciles к local active profile
