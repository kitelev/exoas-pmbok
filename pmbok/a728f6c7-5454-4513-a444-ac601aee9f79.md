# Lesson — Strategy pivot к orchestrator-direct after 3 silent rate-limit child deaths

> **Closure:** [[d90daecb-9c96-440b-9a68-e1c2237d8da8|Closure — RFC 0a0791c1 — FocusProfile Phase 4]]
> **Category:** [[pmbok__LessonCategoryTechnical]]
> **Captured at:** 2026-06-01T02:06:00 (Asia/Almaty)

## Insight

При orchestration multi-task project — после **2+ silent rc=1 child Claude deaths within session** root-caused на Anthropic API rate-limit at orientation phase, **strategy pivot к orchestrator-direct execution** для remaining tasks является cost-effective mitigation:
- **Eliminates rate-limit blast radius** — single API stream вместо parallel/rapid-fire concurrent streams
- **Saves spawn overhead** — child orientation costs ~$20-50 per task in failed cases + retry burn
- **3-5x faster** для small/mechanical tasks (≤200 LOC, stub, config tweak, vault edit)
- **Quality often higher** — B.6 PluginLockManager shipped 100% coverage SEC-001 compliant в 6 min orchestrator-direct vs typical 15-30 min child spawn

## Context

RFC 0a0791c1 FocusProfile Phase 4 (2026-05-31/2026-06-01) — orchestrator session наблюдал **3 silent child deaths за 4h window**:
- B.2 attempt 1 (claude-child-5316cdce-1780253507): silent rc=1 after 11 min orientation
- B.3 (claude-child-dd45de4c): silent rc=1 post-PR-open, before code-reviewer round (resurrected via direct merge based on CI 34/34 success)
- B.4 attempt 1: silent rc=1 at orientation

Pattern detected at 2026-06-01T02:06:00. Strategy pivot announced via PROGRESS ping. Subsequent 8 tasks (B.6/B.5/B.4/B.7/B'.1/B'.2/B.8/B.10) shipped orchestrator-direct без any further silent deaths.

## Applicability

**Trigger conditions:**
- 2+ silent rc=1 child deaths within session
- 3+ concurrent Opus 4.7 streams consuming substantial tokens
- Rapid spawn sequence (≥3 spawns within 30 min)
- Heavy task saturation (~400 LOC + advisor + code-reviewer)

**Decision tree для individual task:**
- LOC ≤200 + mechanical (stub, config, vault edit) → ALWAYS orchestrator-direct
- LOC ≥400 + security-critical (PAT/crypto/auth) → still child IF rate-limit cleared (≥30 min между spawns)
- LOC ≥400 + low-security → orchestrator-direct (sacrifice code-reviewer round для recovery)

**Trade-offs:**
- ⚠️ Orchestrator context grows linearly (10 tasks inline ≈ +250K tokens projected)
- ⚠️ No isolated rollback boundary — orchestrator crash loses all in-flight work
- ⚠️ Code-reviewer round-1 skipped → security-critical PRs need separate retroactive audit Issue (cf. B.3 → #3312)
