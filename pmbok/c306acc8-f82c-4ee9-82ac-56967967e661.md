# Lesson — Spawned child Claude crashes rc=1 at high token-density

> **Closure:** [[87e5f86d-998a-4ec8-b10e-b0cf2b196b07|Closure — Homoiconic plugin testing pilot]]
> **Category:** [[ddfef226-2e3a-4c16-8f25-13f5aabbbfb1|pmbok__LessonCategoryTechnical]]

## Insight

Spawned child Claude sessions running Opus 4.7 exit with **rc=1 (not graceful)** around ~100-200K tokens after substantive work completes but before final cleanup steps. Observed twice in one session (Tasks 0.2 and 0.3 children — same RFC v2 implementation work). Pattern:

- Child finishes 80-90% of execution protocol
- Hits ≥1 hook block OR Skill/advisor heavy interaction
- Suddenly exits rc=1 without emitting `COMPLETED` ping or self-`/quit`

Cost impact: each crash required orchestrator manual recovery (~5-10 min wall-clock + token cost to inspect worktree, recover commits, push branch, open PR).

## Context

- **Task 0.2 child** (claude-child-a6d73d4a-1779531571) — crashed after PreconditionEvaluator injection complete, BEFORE tests + commit. ~$2.76 cost up to that point. Caused by 2× consecutive `bash-prefer-dedicated.sh` hook blocks on standalone `find` commands.
- **Task 0.3 child** (claude-child-fb99b850-1779534569) — crashed after `commit + push` BUT BEFORE `gh pr create`. ~$4.76 cost. Cause less clear; child had ~200K tokens used + multiple advisor calls + heavy Skill interactions.

## Applicability

Future orchestrator-spawned children should:

1. **Watch for EXITED rc≠0** in `~/.claude/audit/child-lifecycle.jsonl` — recovery path: orchestrator inspects worktree state (commits often survive crash), pushes branch + opens PR manually.
2. **Reduce child workload per spawn:** prefer 1 atomic task per child vs cramming worktree + multiple PRs. Each child should target <100K tokens budget.
3. **Mechanical/verification tasks** (no code generation): execute directly in orchestrator session — child spawn overhead exceeds value when work is trivial.
4. **High-effort decisional tasks** (advisor-heavy): may still benefit from child isolation — but plan for crash recovery as default path.

## Cross-project

This is the SECOND project where child stability at high token-density was an issue (first was during RFC 31c1a0be Phase 4 — verified empirically via `~/.claude/audit/child-lifecycle.jsonl` historical search). Worth raising as a meta-issue for spawn-nested-claude skill enhancement (graceful exit hooks, token-budget pre-warning).
