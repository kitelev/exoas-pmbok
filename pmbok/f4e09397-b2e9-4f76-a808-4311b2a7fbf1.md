# Phase Charter — Phase 3 CI workflow

Этот Charter покрывает delivery boundary **Phase 3 — CI workflow**. Root baseline — [[ed448e40-26a7-46d7-a419-5aa9fa985390|root Charter]]. PCS 3/4 (subtree weight 4 + distinct soundness AC + budget 8-16h).

## Scope

Local + remote CI runner для прогона test__PositiveCommandTest ассетов:
- `scripts/run-test-cases.sh` — SPARQL query → `apply --seed --frozen-clock` → byte-for-byte diff → exit code. Sanity gate (count Approved == 0 → fail-fast)
- Integrate predictor/harness modules из `command-pilot.integration.test.ts` как diff-oracle library (~15h savings reuse)
- GitHub Actions workflow: fresh clone+submodule init + disable user-side hooks + `validate schema --shapes-mode` + run-test-cases
- Failure-mode reproduction: PR с intentional grounding corruption → CI must turn red

## Deliverables

- D3.1: scripts/run-test-cases.sh
- D3.2: Reusable predictor/harness library
- D3.3: GitHub Actions workflow .github/workflows/abox-test.yml
- D3.4: Soundness gate reproduction documentation

## Phase-specific AC (soundness — distinct from root)

- [ ] Local script green on pilot test
- [ ] CI green на push в master
- [ ] CI wall-time ≤ 5 min для 1 test-case
- [ ] **Soundness gate verified:** PR с corrupted grounding → CI status = failure within 1 run, с человеко-читаемым diff
- [ ] Sanity gate: rename Approved → Proposed → script returns non-zero exit (prevents silent zero-test green)

## Phase budget

8-16h. Cost drivers: bash runner ~3h + predictor library reuse ~3h + GitHub Actions ~3h + soundness PR test ~2h.
