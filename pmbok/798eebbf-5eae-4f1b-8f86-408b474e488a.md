## 2. Business Case

iPhone vault reindex занимает до 10 минут после Obsidian restart (~163K triples, 11K файлов). Memory pressure часто triггерит auto-restart Obsidian → каждый restart = блокирующая 10-мин wait. Параллельно: cognitive load от irrelevant assets в SPARQL/search results, privacy boundary leakage между work/personal contexts во время screen sharing, tight coupling без объявленных imports (SPARQL audit показал 30+ ontology pairs с counts 100-5868 без `exo__Ontology_imports` declarations).

Текущая онтология описывает physical storage organization (AssetSpace/Ontology/Dependency), но не runtime user preference — отсюда нет declarative mechanism для lazy-load.

## 3. Цели (SMART)

- **Specific:** Внедрить TBox `exo__FocusProfile` + 3 properties (`_includes`, `_extends`, `_alwaysOnOverlay`) для homoiconic declaration of user-selected ontology subset.
- **Measurable:** 4 TBox files в `vault-2025/assetspaces/exo/`, 4 proof-of-concept FocusProfile instances в `shared-identities/`, 1 validation hook с cycle+depth checks, 1 CLAUDE.md update + 1 aiKnow memory entry.
- **Achievable:** Migration Engineer estimated 115-160 min AI-orchestrated wall-clock для Phase 1+2+3+6 in-scope.
- **Relevant:** Решает 4 проблемы из RFC (iPhone reindex, cognitive load, privacy, tight coupling) + restores Homoiconicity Q1=Yes compliance (overlay materialized в RDF).
- **Time-bound:** Сегодня (2026-05-31), end-of-day finishing M4.
