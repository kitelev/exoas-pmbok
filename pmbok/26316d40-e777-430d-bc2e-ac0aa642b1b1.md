# R5 — Plugin lock stale after crash

Если plugin crashes mid-switch, persisted lock file `.exocortex/switch-lock.json` remains — preventing future switches. Without auto-recovery, user must manually delete lock file.

Mitigation: B.6 heartbeat + isStale check (5min timeout) + plugin onload recovery routine — auto-completes idempotent re-index OR prompts user если destructive operation в-progress.
