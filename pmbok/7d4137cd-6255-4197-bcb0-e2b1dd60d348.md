# R1 — nanotar bundle compatibility

Tar-stream library (originally proposed) рассчитан на Node-only runtime — depends на `bare-fs` и `streamx` (Bare-runtime), что не работает в Obsidian's browser-like plugin sandbox. Если ship'нем неподходящую tar library — plugin crash при первом pull attempt.

Phase 0 spike caught — switched к nanotar@0.3.0 которая zero-deps ESM + builtin parseTarGzip async iterator. Risk effectively resolved pre-implementation.
