# Lesson — Orchestrator Monitor blind на orchestrator-direct PR merge requires self-ping

> **Closure:** [[d90daecb-9c96-440b-9a68-e1c2237d8da8|Closure — RFC 0a0791c1 — FocusProfile Phase 4]]
> **Category:** [[pmbok__LessonCategoryTechnical]]
> **Captured at:** 2026-06-01T08:35:00 (Asia/Almaty)

## Insight

Когда orchestrator ships PR через **orchestrator-direct execution** (без child spawn — pattern triggered by rate-limit pivot, см. sibling [[a728f6c7-5454-4513-a444-ac601aee9f79]]), Monitor `tail -F` armed только на `tmux/child-lifecycle.jsonl + orch-push-bridge.jsonl + tmux pane content`. GitHub release webhooks **не triggered**. Result: orchestrator wait-loop continues indefinitely после merge — observed **6h stuck state** (B.4 merged 02:24, parent reconciliation required at 08:35).

**Mandatory mitigation:** after every orchestrator-direct merge, explicit self-ping:

```bash
~/.claude/lib/child-ping-progress.sh PROGRESS "PR_MERGED #${PR_NUM} (${SHORT_SHA}) → ${VERSION} published; task ${TASK_LABEL} Done"
```

Self-ping writes к `child-lifecycle.jsonl` → triggers Monitor → wakes orchestrator → resumes next task spawn.
