# Lesson — Charter §10 token-budget calibration tier needed для AI-orchestrated work

> **Closure:** [[d90daecb-9c96-440b-9a68-e1c2237d8da8|Closure — RFC 0a0791c1 — FocusProfile Phase 4]]
> **Category:** [[pmbok__LessonCategoryScopeManagement]]
> **Captured at:** 2026-05-31T23:09:00 (Asia/Almaty)

## Insight

Current PMBOK Charter §10 (Budget) estimates token-cost via single «AI compute estimate» line с **untiered baseline** (typically $0.50-3.00 для vault-only work). Empirically **wrong by 30-100x** for AI-orchestrated code-PR work с security hardening / code-reviewer rounds / advisor consultations.

**Empirical evidence (RFC 0a0791c1):**
- Charter §10 baseline: $0.50-1.50 per Opus pricing
- Actual project total: **~$190** ($130 orchestrator session + ~$60 children combined)
- Per-task averages:
  - Vault-only TBox work (A.1): **<$1** ✓ matches baseline
  - Code-PR-light (B.2 TarExtractor, B.6 PluginLockManager via direct): **$5-15** 5-30x
  - Code-PR-security-hardened (B.0 nanotar pinning, B.1 GitHubRestClient): **$15-45** 30-100x
  - Heavy state-machine class (B.4 FocusProfileSwitchManager via direct): **$10-20** 20x

## Context

Single-tier Charter estimate causes:
- **Misaligned sponsor expectations** — actual cost вызывает surprise post-close
- **Inadequate session-budget alarms** — orchestrator's "$20 single child" threshold from `Autonomy Decision Matrix` triggered repeatedly без being a real budget breach (just normal cost of security-hardened code work)
- **Forecast errors** — EAC calculation flawed (project total / tasks_done × tasks_total = wrong projection если baseline tier per task wrong)

## Applicability

**Proposed tier model для future `/pmbok-init` Charter §10:**

| Task class | Baseline cost | Rationale |
|---|---|---|
| Vault-only TBox/ABox | <$1 | No code edits, no CI, no advisor — pure asset write |
| Code-PR-light (≤200 LOC, no security) | $5-15 | Single advisor + CI + maybe 1 PR comment iteration |
| Code-PR-security-hardened (PAT/crypto/auth/adversarial inputs) | $20-50 | Code-reviewer round-1 mandatory + multi-iteration hardening + adversarial fixture tests |
| Heavy state-machine class (≥400 LOC + DI + lock + journal) | $30-50 | Architecture discussion + advisor multi-round + extensive unit + integration tests |
| Orchestrator session (multi-task coordination) | $30-100 | Monitor task burns ~$3/h idle + active orchestration cost |

**Charter §10 should итерироваться per task in WBS** instead of single project total. Example:
```
- D1 TBox property: <$1 (vault-only)
- D2 GitHubRestClient: $20-50 (security-hardened)
- D3 TarExtractor: $5-15 (code-PR-light)
- ...
Total estimate: $90-200 + orchestrator $30-100 + buffer 20%
```
