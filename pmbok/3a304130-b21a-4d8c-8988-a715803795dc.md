# R3 — Plugin RDF filter doesn't reduce metadataCache reindex time

Obsidian's metadataCache reindexes all physically-present files at startup regardless of plugin RDF filter. Since v3 backward-compat keeps all submodule files physically present, iPhone reindex pain (10-min wait per restart) NOT improved by Phase 4 alone.

Explicit accepted trade-off для incremental shipping. Phase A+B delivers cognitive load + privacy boundary + REST infrastructure; Phase C+D future RFC delivers physical reindex perf через filesystem removal.
