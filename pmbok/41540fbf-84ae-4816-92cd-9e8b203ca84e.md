# Lesson — Bash `find`/`grep` hook blocks destabilize child sessions

> **Closure:** [[87e5f86d-998a-4ec8-b10e-b0cf2b196b07|Closure — Homoiconic plugin testing pilot]]
> **Category:** [[285d7a99-ae93-4234-a224-c9e168bf2547|pmbok__LessonCategoryProcess]]

## Insight

`~/.claude/hooks/bash-prefer-dedicated.sh` blocks standalone bash `find` and `grep` commands per the `prefer-dedicated-tools` memory rule. A **single block is recoverable** (Claude reads the rule, switches to Glob/Grep tools). But **2 consecutive blocks on similar commands** correlates with child Claude session exit rc=1 — the child appears to enter an unrecoverable state.

**Empirical:** Task 0.2 child crashed exactly 0:32 after the 2nd bash `find` was blocked (no further user activity, no successful tool call between the block and EXITED ping in `child-lifecycle.jsonl`).

## Mitigation

**Child prompt template should EXPLICITLY instruct:**

> ⛔ КРИТИЧНО: Tool discipline (avoid prior child crash)
>
> НЕ использовать bash `find` / `grep` standalone — pre-tool hook `bash-prefer-dedicated.sh` BLOCKS их и сессия может crash при повторе.
>
> | Задача | ✅ Используй | ⛔ НЕ используй |
> |---|---|---|
> | Найти файлы по pattern | Glob tool | `find <path> -name <p>` |
> | Найти текст в файлах | Grep tool | `grep -rn <p> <path>` |
> | Bash OK когда | composite pipe (`cat x \| jq`), heredoc, npm/git/gh CLI | standalone find/grep |

This was added to Task 0.3 child prompt → child successfully avoided bash `find` for the entire session (used `Read` tool + composite Bash with pipes when filesystem traversal needed).

## Process implication

**Spawn-nested-claude skill enhancement candidate:** auto-prepend this tool-discipline block to all child prompts (orchestrator-spawned children specifically). Trade-off: adds ~500 tokens to every spawn prompt vs prevents ~$5-50 child crash recovery cost.
